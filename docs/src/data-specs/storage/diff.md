# DIFF/DIFN - 蓄積系ソフト用 蓄積情報

## 基本情報

- **データ種別**: 蓄積系
- **利用可能オプション**: 
  - `option=1`: 蓄積系データ（通常はこちらを使用）
  - `option=3`: セットアップデータ
  - `option=4`: ダイアログ無しセットアップデータ（初回のみダイアログ表示）

## DIFFとDIFNの違い

2023年8月8日以降、拡張版DIFNではフィールドサイズが拡張されました。新規開発ではDIFNの使用を推奨します。

拡張内容：
- 繁殖登録番号のサイズ拡張
- 生産者コードのサイズ拡張  
- 生産者名のサイズ拡張

## データ更新タイミング

| 曜日 | 時間 | 内容 |
|------|------|------|
| 月曜 | 14:00頃 | 前回提供分から変更・追加のあった蓄積情報 |
| 木曜 | 20:00頃 | 前回提供分から変更・追加のあった蓄積情報 |

データベースを最新情報に保つための差分データが提供されます。

## 提供期間

- **提供単位**: 提供日単位（変更・追加のあった蓄積情報全て）
- **保存期間**: 1年間
- **セットアップデータ収録期間**:
  - UM: 1986年以降（競走馬の基本情報）
  - KS: 全件（騎手マスタ）
  - CH: 全件（調教師マスタ）
  - BR: 1986年以降（生産者情報）
  - BN: 1986年以降（馬主情報）
  - RC: 1986年以降（コースレコード履歴、2003年以降のG1レコード履歴）

## 注意事項

1. **差分更新の管理**: データ区分の確認が重要（1=新規、2=更新、0=削除）
2. **地方・海外レース情報**: RA、SEレコードには地方競馬・海外国際レースの情報も含まれます
3. **マスタ情報の整合性**: 各マスタ情報は相互に関連しているため、更新順序に注意
4. **fromtime指定**: 終了日時の指定はできません（開始日時のみ）
5. **FROMタイムの管理**: 前回取得時刻を正確に記録し、取得漏れを防ぐため重複時間を設定
